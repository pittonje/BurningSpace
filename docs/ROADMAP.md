# Roadmap

## Phase 0: Structured Repository

Definition of Done:

- stable multiplayer core is tagged;
- design and architecture documents exist;
- CI runs `npm test`;
- agent instructions are available;
- user approves the persistent-game direction.

## Phase 1: Account Architecture Spike

Definition of Done:

- authentication strategy is selected;
- database strategy is selected;
- session/account boundary is designed;
- no economy implementation yet;
- no production credentials are committed.

## Phase 2: Static Content And Hangar Read Model

Definition of Done:

- static ship class and hull definitions are designed;
- persistent hangar model is approved;
- starter Miner exists as data design;
- Shipyard read model is designed;
- no purchase flow without transaction design.

## Phase 3: Mining And Tutorial Progression

Definition of Done:

- server asteroids are designed and implemented;
- safe starter mining zone exists;
- resource collection is server-authoritative;
- tutorial milestones persist;
- buying the first Scout is possible through server transactions.

## Phase 4: Frontline Creep Waves

Definition of Done:

- creep wave architecture is implemented;
- group AI is budgeted;
- frontline rewards are server-authoritative;
- load tests cover expected NPC counts.

## Phase 5: Hired Pilots

Definition of Done:

- pilot ownership persists;
- behavior presets exist;
- offline grace rule is implemented;
- return-to-base behavior is tested;
- no unlimited offline farming.

## Phase 6: Wing System

Definition of Done:

- player can deploy a limited wing;
- formation and follow commands work;
- human and pilot ships use common runtime systems;
- Fighter unlock is integrated.

## Phase 7: Strategic Objectives

Definition of Done:

- objectives affect frontline pressure;
- balance squads can fill missing population;
- additional classes are designed;
- faction-level progress is visible.

## Phase 8: Production Hardening

Definition of Done:

- monitoring exists;
- economy audit paths exist;
- load testing is regular;
- security review is complete;
- deployment strategy is approved.
