# Core Loop

## Micro Loop

The moment-to-moment loop:

```text
control
-> combat or mining
-> resource or tactical result
-> return / delivery
-> repair or next sortie
```

This loop must remain responsive and readable. Combat authority stays on the server, and the client remains responsible for presentation, input, interpolation, and UI.

## Medium Loop

The account session loop:

```text
earn
-> buy a ship
-> hire a pilot
-> configure a wing
-> unlock a new role
```

This loop should encourage returning to the hangar between sorties, comparing ships, assigning pilots, and choosing the next objective.

## Long Loop

The long-term progression loop:

```text
develop hangar
-> expand wing
-> unlock classes
-> form squadron
-> participate in the large-scale front
```

The long loop is persistent and account-bound. It should unlock strategic options, not create irreversible domination.

## Team Front Loop

The shared faction loop:

```text
NPC waves
-> frontline clash
-> player and wing participation
-> pressure changes
-> capture or loss of objects
-> pressure on enemy base
```

This loop creates the feeling of an ongoing war even when few human players are present.

## Victory

The final victory mechanic is not decided. Options include persistent territorial control, campaign resets, seasonal objectives, base pressure, strategic events, or no permanent final victory.
