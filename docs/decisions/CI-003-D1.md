# CI-003-D1 — Deterministic trusted-base PR-risk routing for Claude QA

Status: `accepted`

Date: `2026-07-14`

Owner: `Product Architect`

Scope / domain: `CI governance — PR QA routing and QA evidence binding`

## Decision

PR QA routing is determined by a deterministic classifier executed only from the pull request's trusted base commit, never from pull-request-head code. The classifier consumes the pull request changed-file list as JSON data and emits only bounded fixed-vocabulary outputs, never raw paths.

Retrieval, classifier, schema-validation, and stale-PR-state failures fail closed to QA-required behavior. Supported fail-closed reason codes include retrieval_error, classifier_error, and stale_pr_state. Classifier-internal empty_change_set and malformed_path conditions also require QA.

Only the literal qa_required=false output skips the gated Claude QA pipeline steps. A QA skip is a routing outcome, not approval. An approving QA verdict requires validated QA evidence bound to the reviewed commit.

Before QA evidence is published, the workflow binds the review to the reviewed commit and requires reviewed_commit to match the pull request head SHA. Before comment publication, the workflow re-checks the live pull request head, open state, and draft state, and skips superseded runs.

Manual reviewer suggestions are advisory routing outputs. Human merge authority is unchanged.

## Rationale

The routing architecture prevents a pull request from using its own modified workflow or classifier to avoid review. Fail-closed behavior prevents retrieval, classifier, schema, and stale-state failures from silently skipping QA. Commit binding keeps QA evidence attributable to the exact reviewed head.

## Consequences

- Documentation-only or test-only changes may skip Claude QA only when the trusted-base classifier emits the literal qa_required=false outcome.
- Governance, agent, handoff, workflow, dependency, and production paths require QA unless the accepted classifier contract says otherwise.
- Failures in retrieval, classification, schema validation, or stale-state checks do not silently skip QA.
- Classifier output does not grant merge authority.
- A green check alone is not review approval.
- Manual reviewer suggestions do not by themselves create a merge requirement or approval.
- Workflow changes cannot self-certify through modified pull-request-head workflow code.
- This record does not change workflow behavior or fix existing workflow bugs.
