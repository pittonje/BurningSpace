# BurningSpace

BurningSpace - браузерный космический прототип на Phaser 3, TypeScript, Vite и Colyseus.

Проект использует монорепозиторий. Локальный боевой прототип и сетевой прототип существуют как отдельные сцены: локальная аркада остается клиентской, а новая multiplayer-сцена показывает серверно-авторитетные корабли без боя.

## Структура

```text
apps/client      Phaser/Vite клиент и игровые ассеты
apps/server      Node.js/TypeScript/Colyseus сервер
packages/shared  общие типы, константы и названия сетевых сообщений
```


## Базовый сетевой тест

1. Запустить сервер: `npm run dev:server`.
2. Запустить клиент: `npm run dev:client`.
3. Открыть две вкладки клиента.
4. Нажать `Connect` в обеих вкладках.
5. Задать разные ники.
6. Выбрать разные режимы или фракции.
7. Нажать `Apply profile`.
8. Проверить одинаковый список участников.
9. Закрыть одну вкладку или нажать `Disconnect`.
10. Проверить удаление участника во второй вкладке.

## Сетевые корабли без боя

BattleRoom синхронизирует `participants` и `ships` через Colyseus Schema. Сервер является единственным источником истины для существования корабля, позиции, скорости и поворота. Клиент отправляет только `playerInput`: флаги движения, `aimAngle` и `sequence`.

Player-профиль с faction `red` или `blue` получает один серверный корабль. Spectator подключается к комнате и видит корабли, но не получает собственный корабль и не отправляет input. Серверная симуляция работает с частотой 20 Hz. Клиентская `MultiplayerGameScene` отображает корабли как визуальные представления `ShipState` и интерполирует их позиции между серверными обновлениями.

Бой пока не входит в сетевую сцену: нет стрельбы, урона, здоровья, астероидов, NPC, столкновений и респавна. Кнопка `Open local prototype` запускает отдельную локальную Phaser-сцену, которая не является сетевой.

Сценарий трех вкладок:

1. Первая вкладка: `Connect`, выбрать `player/red`, `Apply profile`, затем `Enter multiplayer`.
2. Вторая вкладка: `Connect`, выбрать `player/blue`, `Apply profile`, затем `Enter multiplayer`.
3. Третья вкладка: `Connect`, выбрать `spectator`, `Apply profile`, затем `Enter multiplayer`.
4. Red и blue игроки управляют только своими кораблями.
5. Spectator видит оба корабля, но двигает только свободную камеру.

Управление в сетевой сцене:

- `WASD` или стрелки - движение игрока или свободной камеры spectator.
- Мышь - направление поворота корабля игрока.
- `Esc` - вернуться в сетевое lobby без отключения от комнаты.
